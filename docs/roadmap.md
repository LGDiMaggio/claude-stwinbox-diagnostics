# Roadmap

This roadmap reflects current priorities. Items are directional and may be reprioritized based on contributor availability and validation results.

## Near term

- Improve reproducibility of setup and troubleshooting docs across OSes.
- Expand automated tests for core DSP and fault-indicator helpers.
- Add more reference examples with known signal characteristics.
- Strengthen MCP tool error handling and parameter validation.

## Mid term

- Improve baseline/trend comparison workflows for repeated acquisitions.
- Expand fault-library coverage while keeping assumptions explicit.
- Improve ergonomics for DATALOG2 workflows in both MCP servers.
- Add release hygiene improvements (versioning, changelog discipline, automation).

## Longer term

- Additional board/sensor compatibility evaluations.
- Better integration guidance with external observability stacks.
- Structured benchmark datasets for comparing diagnostic prompts/workflows.

## Out of scope

- Claims of production-ready autonomous diagnostics.
- Vendor-specific lock-in features not aligned with open MCP workflows.
- Safety-critical decision automation without human review.

## How to contribute to roadmap items

1. Open/locate an issue linked to the roadmap topic.
2. Propose implementation and validation plan.
3. Keep PRs small and evidence-backed.
